# AI漫剧 Agent 网页端 - 需求评审汇报

## 1. 本次评审目标

本次产品需求评审，仅聚焦当前版本需要对齐的 3 个结构模块与 2 个核心功能：

1. 功能区
2. 侧边栏
3. 标签页
4. 分镜图片生成
5. 视频生成

本轮评审范围之外的旧版能力，不作为本次功能承诺内容。

---

## 2. 当前评审版的产品结构

### 2.1 页面结构

当前版本已收口为三段式结构：

1. 左侧：侧边栏
2. 中间：标签页工作区
3. 标签页内容：按功能独立承载具体生成能力

### 2.2 标签页能力拆分

本次已从旧版统一 AIPane 模式切换，拆分为两个独立功能标签页：

1. 分镜图片生成标签页
2. 视频生成标签页

拆分后的产品语义是：

1. 打开“分镜图片生成”就是图片生成功能页
2. 打开“视频生成”就是视频生成功能页
3. 不再通过一个 AI 助手页内部切换图片/视频模式

## 4. 核心实现说明

### 4.1 分镜图片生成标签页

图片生成标签页已具备独立实现能力：

1. 以图片文件为绑定对象
2. 在当前标签页内部完成图片生成
3. 支持图片参数配置
4. 支持生成结果版本记录
5. 支持当前版本选择

### 4.2 视频生成标签页

视频生成标签页已具备独立实现能力：

1. 以视频文件为绑定对象
2. 在当前标签页内部完成视频生成
3. 支持视频参数配置
4. 支持生成结果版本记录
5. 支持当前版本选择

### 4.3 和旧版的本质区别

旧版：

1. 一个 AIPane
2. 里面切换图片 / 视频 / 脚本 / 音频

当前评审版：

1. 图片标签页就是图片标签页
2. 视频标签页就是视频标签页
3. 页签职责与用户理解一致

## 6. 明天评审建议汇报口径

建议你明天这样讲：

1. 这一版先收敛成一个清晰的工作区结构：侧边栏 + 标签页 + 功能区。
2. 标签页层面，本轮只保留两个真正可用的业务能力：分镜图片生成、视频生成。
3. 原来旧版把多种能力混在一个 AI 助手页里，现在已经拆成按标签页承载的独立功能。
4. 其它能力本轮不作为评审目标，后续再按优先级补充。
